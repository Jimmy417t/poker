一、動機與目標
動機：
在日常文書處理或程式開發中，拼字錯誤是常見的問題。現有的編輯器拼字檢查背後的核心技術——Trie（字典樹）與 編輯距離演算法——正是資料結構與演算法的經典應用。本專題希望實作這些技術，深入理解字串處理與記憶體管理的運作機制。
目標：
從零實作 Trie 資料結構，支援單字的插入、搜尋與前綴查詢
實作前綴自動完成功能，給定前綴後回傳最相關的單字建議
實作拼字錯誤修正功能，基於編輯距離（Levenshtein Distance）提供候選單字
確保程式無記憶體洩漏，展現對 C++ 動態記憶體管理的掌握

二、預期功能
單字管理	    支援從字典檔載入單字，或手動插入新單字
單字查詢	    輸入單字，回傳是否存在於字典中
前綴自動完成	  輸入前綴（如 "ap"），回傳所有符合的單字（如 apple, application），可限制回傳數量
拼字錯誤修正	  輸入錯誤單字（如 "appple"），回傳編輯距離最小的正確單字建議（如 apple, apply）
互動式介面	  命令列選單，使用者可選擇功能並輸入內容

三、使用技術
程式語言      C++17
核心資料結構   Trie（字典樹）、Min-Heap（最小堆積）
演算法        Levenshtein Distance（動態規劃）、前綴搜尋
記憶體管理     手動 new/delete、解構式中遞迴釋放節點
輸入輸出      標準輸入輸出、檔案讀取（ifstream）

四、時程規劃
第1週	Trie 核心實作（insert, search, startsWith）	可插入與查詢單字的基本程式
第2週	前綴自動完成功能（autoComplete, Top-K）	輸入前綴後回傳候選單字
第3週	拼字錯誤修正（編輯距離計算與建議）	輸入錯誤單字後回傳正確建議
第4週	整合介面、檔案載入、記憶體檢測與報告撰寫	完整可展示的命令列程式

五、與課程的關聯
動態記憶體管理   	TrieNode 的 new/delete，解構式遞迴釋放
指標與鏈結結構   	TrieNode 中的 children 指標陣列
遞迴           	Trie 的走訪與節點刪除
字串處理	        編輯距離演算法（動態規劃）
時間複雜度分析	    插入/搜尋 O(L)，編輯距離 O(L1×L2)
空間複雜度分析	    Trie 的節點數與字典大小的關係
