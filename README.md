# 迷宮生成與路徑視覺化系統

## Proposal Report

### 動機與目標

**動機**：  
迷宮問題是圖論與搜尋演算法的經典應用場景。從隨機迷宮的生成，到利用不同演算法尋找路徑，背後涉及 Stack（DFS）、Queue（BFS）等核心資料結構的實際運用。本專題希望透過視覺化方式，讓抽象的演算法變得具體可見，同時深入理解不同搜尋策略的差異。

**目標**：  
- 實作高複雜度隨機迷宮生成演算法（具備多分岔與迴路）
- 實作 DFS 演算法，找到一條從起點到終點的路徑
- 實作 BFS 演算法，找到最短路徑
- 實作 Dijksta 演算法，找到成本最小的路徑
- 以彩色文字介面清晰展示迷宮與求解結果
- 比較三者在路徑長度、執行時間、成本上的差異

---

### 競品比較

| 項目 | 傳統迷宮遊戲 | 本專題 |
|------|-------------|--------|
| 迷宮結構 | 單一路徑，無分岔 | 多分岔 + 隨機捷徑，高複雜度 |
| 求解演算法 | 僅有一種 | 三重演算法 |
| 路徑視覺化 | 無或單色 | 彩色區分三種路徑 |
| 複雜度指標 | 無 | 分岔點數量統計 |
| 效能分析 | 無 | 路徑長度、執行時間、成本比較 |

---

### 預期功能

| 功能模組 | 說明 |
|---------|------|
| 迷宮生成 | 隨機生成具分岔與迴路的高複雜度迷宮 |
| 大小自訂 | 使用者可輸入迷宮尺寸（奇數，建議 21~41） |
| DFS  | 使用 Stack 找出一條從起點到終點的路徑 |
| BFS  | 使用 Queue 找出最短路徑 |
| Dijkstra | 在BFS基礎上 引入成本概念
| 彩色顯示 | 牆壁、路徑、起終點以不同顏色區分 |
| 統計數據 | 顯示迷宮分岔數、路徑長度、演算法執行時間 |

---

### 使用技術

| 技術項目 | 說明 |
|---------|------|
| 程式語言 | C++17 |
| 核心資料結構 | Stack (DFS)、Queue (BFS)、二維陣列 (vector) |
| 演算法 | Recursive Backtracking、DFS、BFS、Dijkstra |
| 視覺化 | ANSI 彩色文字輸出 |
| 效能測量 | `<chrono>` 計時 |
| 隨機機制 | `<random>` 洗牌與隨機分岔 |

---

### Prototype 預計可驗證內容

| 驗證項目 | 預期結果 |
|---------|---------|
| 迷宮生成正確性 | 起點到終點存在至少一條路徑 |
| 迷宮複雜度 | 分岔點數量 > 30（以 31x31 為例） |
| DFS 可行性 | 能找出一條路徑到達終點 |
| BFS 最優性 | BFS 路徑長度 ≤ DFS 路徑長度 |
| BFS 效能 | 迷宮越大，BFS 執行時間顯著增加 |
| 視覺化清晰度 | 牆壁、路徑、演算法路線可清楚區分 |
